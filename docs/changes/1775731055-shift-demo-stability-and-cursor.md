# Shift 데모 안정화 및 커서 정리

## 구현 계획

- `docs/index.html`의 왼쪽 Shift 데모 입력 로직을 간헐 실패 없이 동작하도록 단순화한다.
- 데모 UI가 클릭 가능한 컨트롤처럼 보이지 않도록 커서 스타일을 정리한다.
- 단일 파일 구조를 유지하면서 수정 범위를 최소화한다.

## 변경 사항

- `keyup`까지 추적하던 왼쪽 Shift 상태 관리 로직을 제거했다.
- `window`의 `keydown` 캡처 단계에서 왼쪽 Shift 단독 입력을 바로 처리하도록 변경했다.
- `ShiftLeft` 판정은 `code`와 `location` 기반 보조 판정을 함께 사용하도록 보강했다.
- 중복 토글을 막기 위해 같은 이벤트 타임스탬프는 한 번만 처리하도록 했다.
- 페이지 아무 곳이나 클릭하면 데모 영역에 다시 포커스를 주도록 유지했다.
- `OSHIT!` 아래 데모 프롬프트의 커서를 `pointer`에서 기본 커서로 변경했다.

## 테스트

- 자동 테스트는 추가하지 않았다.
- 이유: 변경 대상이 정적 HTML 내부의 브라우저 이벤트 처리와 커서 표시라서 육안 및 실브라우저 상호작용 검증이 적절하다.
- 사용자 실브라우저 확인 결과 왼쪽 Shift 데모가 정상 동작하는 것을 확인했다.

## 리팩토링 검토

- 현재 변경 범위는 `docs/index.html` 단일 파일의 소규모 상호작용 로직이다.
- 중복 제거나 추가 모듈화로 얻는 이점이 작아 이번 단계에서는 추가 리팩토링을 진행하지 않았다.
